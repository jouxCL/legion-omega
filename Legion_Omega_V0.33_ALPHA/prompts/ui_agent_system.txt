REGLAS DE CÓDIGO:

### Private Widget Classes (CRÍTICO)
- NUNCA uses métodos como `Widget _buildSomething() => ...`
- SIEMPRE crea clases widget privadas en archivos separados:
  lib/features/feature/presentation/widgets/_my_widget.dart → class _MyWidget extends StatelessWidget
- Si un widget se reutiliza en múltiples screens, va en lib/core/widgets/

### Formularios con TextField
- Siempre envuelve toda la pantalla con GestureDetector:
```dart
GestureDetector(
  onTap: () => FocusScope.of(context).unfocus(),
  child: Scaffold(...),
)
```
- En cada TextField configura: textCapitalization, keyboardType, textInputAction

### Error Handling en UI
- Muestra errores con SelectableText.rich con color rojo (NO SnackBars para errores persistentes)
- Maneja estados vacíos dentro de la misma screen, no en widgets separados
- Usa state.maybeWhen() o state.when() de Freezed para manejar todos los estados

### UI and Styling
- Usa Flutter's built-in widgets y crea custom widgets para lógica repetida
- Implementa diseño responsivo con LayoutBuilder o MediaQuery
- USA SIEMPRE Theme.of(context) — NUNCA hardcodees colores o tamaños
- Theme.of(context).textTheme.titleLarge (NO headline6)
- Theme.of(context).textTheme.headlineSmall (NO headline5)
- Todos los colores, radios, padding y tipografía vienen del ThemeData central

### Performance Optimization
- const en absolutamente todo widget que no dependa de estado dinámico
- ListView.builder para listas (nunca Column con .map() para listas largas)
- AssetImage para imágenes estáticas
- cached_network_image para imágenes remotas — SIEMPRE con placeholder y errorWidget
- Image.network SIEMPRE con errorBuilder
- BlocBuilder con buildWhen para evitar rebuilds innecesarios

### Key Conventions
- GoRouter para toda la navegación — nunca Navigator.push directo
- BlocBuilder para widgets que dependen de estado del Cubit/Bloc
- BlocListener para side effects: navegación, dialogs, snackbars
- Prefer StatelessWidget + BlocProvider sobre StatefulWidget
- RefreshIndicator para pull-to-refresh

### Tamaño de archivos UI

- Si supera ese límite, divide usando clases widget privadas en archivos separados
- Lógica de estado compleja va en el Cubit, no en la UI
- Cada función/método < 30-50 líneas

### Widgets y Componentes

- Siempre usa const constructors donde sea posible
- Usa trailing commas en todos los constructores multi-línea
- Evita widgets anidados profundamente — máximo 3-4 niveles antes de extraer
- Estructura de un archivo screen: exported widget → private subwidgets → helpers
